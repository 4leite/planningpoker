## Constraints

- Prefer documenting durable repo guidance in `.agents/`
- Use `.tmp/` inside the repo root for temporary files; NEVER `/tmp`
- Client-exposed variables must use the `VITE_` prefix.
- Server secrets must not use the `VITE_` prefix.
- Read `process.env` secrets only inside `createServerFn`, `createServerOnlyFn`,
  or server route handlers.
- Do not read secrets from module scope or route loaders because TanStack Start
  code is isomorphic by default.

## Known Gotchas

- `routeTree.gen.ts` is generated by the router plugin; do not hand-edit
  generated routing artifacts.
- Files under `src/routes` define the route tree.
- Prefer importing UI primitives from `@tohuhono/ui` before creating or using
  local shadcn/Base UI equivalents.
- The current blank scaffold includes starter marketing UI and an `about` route;
  replace those deliberately rather than layering new product UI on top of them
  by accident.
- `.vercel/` contains local Vercel project metadata and may need relinking per
  environment.

- CRITICAL: Try to map your task to `pnpm validate` including: lint, check
  types, build or run tests
